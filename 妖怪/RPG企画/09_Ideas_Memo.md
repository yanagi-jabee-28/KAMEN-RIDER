# 09 Ideas Memo | アイデア・没案メモ

このドキュメントは、企画会議の過程で出たが最終的に採用されなかったアイデアや、名称変更の履歴を記録するためのものです。

## 1. 職業名の変遷

-   **指揮役:**
    -   `軍師` → `軍主` → `兵法者` → `兵法家` → **`兵部少輔（通称：軍師）`** に決定。
    -   **理由:** 「軍主」は一般的でなく、「兵法者」は武芸者のニュアンスが強いため、「兵法家」へ。最終的に、古代の官職名である「兵部少輔」とし、通称を「軍師」とすることで、キャラクターの背景とプレイヤーの分かりやすさを両立させた。
-   **速攻役:**
    -   `斥候` → **`行者`** （イベントによるクラスチェンジ）に決定。
    -   **理由:** 単なる狩人から、役小角をモチーフにした修験者へと成長するドラマを描くため。
-   **魔法役:**
    -   `魔術師` → `外法師` or `陰陽師` → **`外法師`と`陰陽博士`の二人体制**に決定。
    -   **理由:** 攻撃特化と万能支援の役割を明確に分離し、師弟関係というドラマ性を追加するため。「厭魅」という名称案は不採用。
-   **勇者役:**
    -   `勇者` → `浪人` → **`稀人（ミコト）`** に決定。
    -   **理由:** 物語の核心である「神の器」という設定を反映させるため、「外から来た謎の人物」という意味合いを持つ「稀人」を採用。

## 2. ボス設定の変遷

-   **ラスボス:**
    -   当初案: `ヤマタノオロチ`（巨大な怪物）
    -   決定案: **`ヒルコ`（ミコトの失敗作の兄）** が操る`ヤマタノオロチ`（機械要塞）
    -   **理由:** 単なる怪物退治で終わらせず、主人公の出自に関わる「兄弟対決」というドラマチックな構造にするため。
-   **裏ボス:**
    -   当初案: `ヒルコ`
    -   決定案: **`別天津神`（世界の管理者AI）**
    -   **理由:** 物語のSF設定（オーパーツの根源）をクリア後シナリオで完全に開示し、本編の神話的雰囲気を壊さないようにするため。

## 3. 連携技の初期案

-   当初は、特定のメンバーの組み合わせでのみ発動する技が中心だった。
-   **改善:**
    1.  いつでも誰とでも使える**汎用4人連携『四魂の陣』**を導入し、パーティ編成の自由度を確保。
    2.  特殊状態**『神懸かり』**を導入し、強力な連携技を「必殺技」として差別化。
    3.  同じペアでも発動技が変わる**『阿吽システム』**を考案。

## 4. 検討したが不採用となったアイデア

-   **ミコトの能力:**
    -   初期案では「アイテムマスター（道具の効果2倍）」という案もあったが、「剣士なのになぜ薬草に詳しいのか」という違和感から不採用に。最終的に「神写し（コピー）」能力に集約された。
-   **陰陽師の役割:**
    -   当初は役割が曖昧で「裏方に押し込まれている」感があった。
    -   **改善:** 「五行属性の専門家」「デバフのスペシャリスト」という明確な役割を与え、外法師との師弟関係を設定することで個性を確立した。
-   **SF要素の度合い:**
    -   「レーザービーム」や「ロボット」のような露骨なSFではなく、「火縄銃」「湾刀」「からくり人形」といった、**「数百年先の技術」**に留めることで、歴史ファンタジーとしての品格を保つ方針に決定。

## 6. アイデア採録：『アシブネノミコト —The Vessel of Genesis—』

### 概要 🔧
-  主題: 神話的な「国造り（救世）」とSF的な「廃棄（国譲り）」を主語ごと反転させるミスリード。
-  キーアイデア: タイトル／主人公／システムの「意味」をプレイヤーが後から180°認識し直す構造（寄生獣的主語反転 + DQ6的世界反転）。

### コア反転（短く整理） ✅
1.  プレイヤーの前提（表）: 「ミコト＝救世主／アシブネ＝聖なる葦船」
2.  真実（裏）: 「ミコト＝廃棄プロセスの実行体／アシブネ＝廃棄（フォーマット）プロセス名」
3.  構造的二段階ラスボス: ヒルコ（感情の決着）→ 別天津神（システムの正体）

### 主要設定の再定義（実装に直結するポイント） ⚠️
-  神写し（ゲーム内スキル）＝『サンプリング＆デリート』プロトコル（使用頻度で仲間の“実体”が劣化→最終的に世界から削除される）。
-  アシブネ＝システム名（例: ASHIBUNE-01）＝地上データ回収→ローカル削除を実行する“廃棄モード”。
-  ヒルコ＝最初の“失敗した回収個体”であり、地上に情を残したために排除を拒んだ反乱者。

### ゲーム演出・システム案（具体案） 💡
-  UI表現: 重要な収集/開眼イベント後に「Upload Complete / Delete Local Source」のフラッシュ表示。
-  隠しステータス: `siphon` カウンター。神写し使用で上がり、閾値到達で仲間が永続的に能力低下→イベントで“存在消失”。
-  ドラマ演出: ヒルコ戦で“感情的決着”を与え、倒した瞬間に衛星AIのログが流れて真の目的が明かされる（伏線回収）。
-  エンディング分岐: ヒルコに協力してシステムを破壊（人間性保持）／ヒルコを倒してシステムを起動（全データ回収＝文明リセット）。
