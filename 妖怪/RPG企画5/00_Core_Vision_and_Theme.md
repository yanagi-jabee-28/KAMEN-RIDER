# RPG企画5 - 00 Core Vision and Theme (思想と基盤)

本ドキュメントは、RPG企画5における不変の憲法である。仕様追加・変更・削除の議論は、必ず本書の規約に従って実施する。

## 1. プロジェクト基本情報
- **プロジェクトコード:** RPG企画5
- **タイトル:** アシブネノミコト 〜泥と星屑の還り路〜
- **ジャンル:** 和風神話ダークファンタジーRPG
- **コア・エクスペリエンス:** 壊れた器を継ぎ、痛みを連れて前へ進む「修復と継承」の旅。

## 2. 核心対立軸 [#ID:R5_Core_Conflict]
**永遠へ凍らせる結晶化** と **傷を抱えて巡る還流** の衝突を、全仕様の唯一の判定軸とする。

- 天の理: 悲劇を再発させないために、変化を停止し保存する。
- 地の理: 破損と死を受け入れ、継ぎ直しながら命を渡す。

## 3. 厳格な仕様変更・依存解決プロトコル [#ID:R5_Traceability_Protocol]
ユーザーから仕様要求を受けた際、必ず以下4フェーズを順守する。

### Phase 1: 影響範囲の特定（Impact Analysis）
以下5層で影響を棚卸しする。
1. 思想・基盤層: 本書のコアテーマと矛盾がないか。
2. 物語・世界観層: `10_Narrative_and_Characters.md` のトラウマ、時系列、環境定義と整合するか。
3. メカニクス・UI層: `20_Game_Systems_and_Flow.md` の戦闘導線、UI、体験設計へ何が波及するか。
4. データ・論理層: `30_Data_and_Logic_Architecture.md` の変数・式・フラグへ何を追加修正するか。
5. アート・演出層: `40_Art_Direction_and_Assets.md` の和紙・岩絵具・金継ぎ表現へどう反映するか。

### Phase 2: 単一情報源（SSOT）の決定
- 詳細仕様は**単一ファイルのみ**に記述する。
- 複数ファイルへ同一詳細を書くことを禁止する。
- 実装値・計算式・フラグ定義のSSOTは原則 `30_Data_and_Logic_Architecture.md` とする。

### Phase 3: アンカーID参照網の構築
- SSOT見出し末尾に `[#ID:Domain_Feature]` を付与する。
- 従属ファイルは「概要 + SSOT参照」のみを記述する。
- 例: `※詳細は 30_Data_and_Logic_Architecture.md の [#ID:R5_Resource_Core] を参照`

### Phase 4: 変更案と波及リストの提示
ユーザーへの提示は必ず以下3部構成にする。
1. 変更の核心（SSOT）
2. 連動して書き換えるべきファイル（波及更新）
3. アーカイブ記録（`99_Archive_and_Changelog.md` 追記案）

## 4. 憲法（絶対遵守事項） [#ID:R5_Constitution]
- **DQ由来要素の直輸入禁止**: 記号や名称をそのまま持ち込まない。
- **SF用語の排除**: 機械語や近未来語で世界観を説明しない。
- **三条熱源の原則**: 活魂・情念・武器耐久度のみを基底リソースとし、新規ゲージ乱立を禁止。
- **違反要求への対応**: 要求を拒絶せず、本作語彙へ翻訳した代替案を逆提案する。

## 5. SSOT運用ルール [#ID:R5_SSOT_Rules]
- 30以外のファイルに計算式、内部変数、進行フラグを記述しない。
- 10/20/40/50は詳細を複写せず、必ずアンカー参照で接続する。
- 旧仕様の痕跡は `99_Archive_and_Changelog.md` に集約し、現行仕様へ逆流させない。
