# 『アシブネノミコト —魂の還流と金継ぎの神話—』 開発プロジェクト

本ディレクトリは、歴史を変える神話のRPG『アシブネノミコト』の企画と仕様を管理する場所です。
この作品は単なるゲームではなく、プレイヤーの心が動く「物語の温かさ」
と、古い時代から続く神話らしい「質感」を何より大切にして
制作されます。開発者は常に、プレイヤーの気持ちと物語がぴったり合うように意識しながら、
アイデアを形作っていきます。
すべての設定、システム、シナリオは、以下の「開発方針」に則り、指定された10のファイルに厳密に分類・蓄積されます。

---

## 🧭 1. プロジェクト開発方針（Core Philosophy）

本プロジェクトにおいて、いかなる新規アイデアを追加する際も、必ず以下の3つの方針（フィルター）を通すこと。

1. **SF的な表現を使わず、神話らしさを貫く**
   * 「システム」「データ」「AI」といったSF的な言葉や考え方は使わない。
   * すべてを「呪術」「天の理」「穢れ」「自然の心」といった言葉で表現して、冷たく感じる恐怖と、命の温かさを対比させる。
2. **物語とゲームのシステムがぴったり合う**
   * 「レベルを上げて強くなる」というRPGの基本的な行動に、「仲間の気持ちや思い出が吸われていく」という代わりの代償をくっつける。
   * プレイヤーが「強くなりたい」という気持ちが、そのまま神話の厳しさ(いけにえ)として感じられるようにする。
3. **テーマの貫徹：「すべてをわかろうとするのは人間の傲慢である」**
   * 他者の魂を管理・保護しようとする神（システム）の傲慢さを打ち破り、傷つきながらも自立して生きる人間たちの「金継ぎ」の物語を描き切る。
4. **魂と感情は熱のようなもの**
   * この世界では「魂・感情・穢れ」は、熱みたいに考える（本当の温度ではなく、比喩）。消えることはなく、必ずどこかに移動していく。
   * 敵を倒した際や、仲間が神格化した際、「その失われた熱（魂）は今どこにストックされているか？」を常に意識し、設定のブラックボックスを作らないこと。
   * どのファイルに書き込むにせよ、魂の循環経路が辻褄を合わせるように意識する。

## 📂 2. ファイル構成と記述定義（Directory Structure）

新しいアイデア、設定、ギミックを思いついた際は、その性質に応じて必ず以下の10ファイルのいずれかに追記すること。
**いつも気をつけることは：**
- 古い設定にSF的な言葉や考え方が残っていないか、時々チェックすること。
- すべての文章は「心が動き、神話らしい」べきで、
   古い言葉や考え方が混ざっていたら、すぐに直すこと。

### 📄 00_Project_Bible.md（企画の土台）
* **何か:** このプロジェクトの「ルール」。ぜったいに変わらないもの。
* **中身:** メインテーマ、キャッチコピー、遊ぶ人たち、全体の雰囲気、このゲームだけの特徴。
* **追加するアイデアの例:** 「新しいキャッチコピー案」「他作品との差別化ポイント」

### 📄 01_World_Mythology.md（世界の決まりと神話の言葉）
* **何か:** ゲームの舞台の法則と、使う神話の言葉をまとめたもの。
* **中身:** 天のルールと地の汚れの関係、古い日本の土地、常世と黄泉の設定、使ってはダメな言葉/使う言葉。
* **追加するアイデアの例:** 「架空の伝承や歴史年表」「NPCの文化風俗」「新しい神話的専門用語」

### 📄 02_Story_Structure.md（物語の流れ）
* **何か:** 物語のあらすじ、進み方、話の分岐をまとめたもの。
* **中身:** 全4つの段階（本編～常世を逆走～本当のED～クリア後）、各章のあらすじ、エンディングの決まり。
* **追加するアイデアの例:** 「特定のNPCのサブクエスト」「中盤のドラマチックな展開案」

### 📄 03_Characters_Souls.md（キャラクターと心の変化）
* **何か:** 登場人物の個性と、心がどう変わっていくかをまとめたもの。
* **中身:** ミコト、ヒルコ、仲間8人の詳しい説明、名前をもらうシーン、各キャラが失っていく心の大事な部分。
* **追加するアイデアの例:** 「仲間の過去の掘り下げ」「敵対するサブキャラクターの動機」

### 📄 04_Ludo_Narrative.md（物語とゲームが合わさる仕組み）
* **何か:** 「物語の面白さとゲームの面白さが一緒」になるように作った、このゲーム独自のシステム。
* **中身:** 『神写し』で心を吸い上げる仕組み、失ったものに耐えるための『心の断片』『つながりの場所』、画面や操作の考え方。
* **追加するアイデアの例:** 「感情を揺さぶるメニュー画面の演出案」「探索中の新しいフレーバーギミック」

### 📄 05_Battle_Skills.md（戦い方と技）
* **何か:** 戦いのルールと、キャラが使う技の数値をまとめたもの。
* **中身:** タイムラインで進む戦いの決まり、3つの段階（人間→神の力→本当の名前）の技の変化、協力技『四魂の陣』、強くなること・弱くなることの説明。
* **追加するアイデアの例:** 「新しい連携技の組み合わせ」「ピーキーで面白い新スキルの効果」

### 📄 06_Enemies_Bosses.md（敵とボス）
* **何か:** 敵の生態と、戦う時の工夫をまとめたもの。
* **中身:** 「地から出た汚れ」「天からの道具」の分け方、**各ボスがこの世界の心の循環の中でどんな役割を持っているか、障害を抱えているか**、カガセオと別天津神の変わった仕掛け、敵を倒した時にもらえるもの。
* **追加するアイデアの例:** 「道中の無気味なザコ敵デザイン」「ボス戦専用の特殊な会話割り込み演出」

### 📄 07_Items_Crafting.md（道具と武具、直し方）
* **何か:** プレイヤーが手に入れて、使ったり、つけたりするすべてのもの。強くする仕組み。
* **中身:** 古い時代の武器や防具、魔法で使うもの、『金継ぎ』で武具を新しく生まれ変わらせる仕組み、ものの説明。
* **追加するアイデアの例:** 「クリア後に手に入るチート武器」「金継ぎに必要な専用素材の設定」

### 📄 08_Level_EndContent.md（マップとクリア後）
* **何か:** 世界がどうつながっているか、クリア後に遊べることをまとめたもの。
* **中身:** マップの繋がり、クリア後だけ行ける『黄泉の裂け目』、『神社（神籬の守り）』の仕組み、隠れたラスボス(禍津日神)。
* **追加するアイデアの例:** 「ダンジョンの謎解きギミック」「神社を巡るスタンプラリー的なやり込み要素」

### 📄 09_Direction_Aesthetics.md（見た目、音、雰囲気）
* **何か:** プレイヤーが目で見て、耳で聞いて、心で感じる「質感」をまとめたもの。
* **中身:** 冷たく、シンプルなアート方針、古い雅楽や自然の音をベースにした音、画面の光の効果（金継ぎの輝きなど）。
* **追加するアイデアの例:** 「絶望的なシーンでのカメラワーク指示」「真名技を放つ際の専用BGMの入り方」

## 💡 3. アイデアの仕分け・分類ルール（迷った時のガイド）

もし新しいアイデアがどのファイルに属するか迷った場合は、以下の基準で判断してください。

* **[数字・ルールか] vs [プレイヤーの気持ちか]**
  * 戦いの計算や技の力なら ➡️ `05_Battle_Skills`
  * その技を使った時、プレイヤーがどう感じてほしいか（画面の工夫など）なら ➡️ `04_Ludo_Narrative`
* **[物語の進み方か] vs [キャラの気持ちか]**
  * 「どこに行って誰を倒すか」というイベントなら ➡️ `02_Story_Structure`
  * 「なぜそのキャラが戦うのか」という心の話なら ➡️ `03_Characters_Souls`
* **[世界に元からあるものか] vs [プレイヤーが手にするものか]**
  * 歴史や土地の成り立ちなら ➡️ `01_World_Mythology`
  * プレイヤーが装備したり使ったりできる物質なら ➡️ `07_Items_Crafting`

## 🚫 4. ライティング時の禁則事項（シナリオ上の注意）

### 「全知の否定」の徹底（テーマの具現化）

* **敵の正体を長く説明させない:**
  * 「この敵は天のルールの一部で……」と NPC が事実を教えることは禁止。
  * 起きたことは「現象」として見せるだけで、プレイヤーの「予感」や「驚き」に任せるほうがいい。
  * 本当のことは、いくつものサブクエストやシーンが積み重なることで初めて、プレイヤーが**自分で気づく**ように設計する。

* **キャラが戦う理由を完全には説明しない:**
  * 敵ボスが「なぜ」ここにいるのかを、すべて説明することは禁止。
  * よく分からないまま戦わせて、倒した後も「あの者は何だったんだろう」という疑問を残す。
  * この「よく分からなさ」が「人間は他の人のことを完全には理解できない」というテーマを、物語そのものに組み込む。

* **イベント画面や動画での「説明」は最小限に:**
  * 映像と音だけで伝わるシーンを大事にする。セリフがない静かなシーンを大切にすること。
  * プレイヤーが「この景色は何を言っているんだろう」と考えて、人それぞれ違う受け取り方をする状態が理想。
  
* **ラスボス（別天津神）は「言葉」を発しない:**
  * 完全な沈黙でプレイヤーに襲いかかる。そのため、プレイヤーはボスの正体・動機・背景をいっさい知らないままただ戦う。
  * 倒した後も語り部の歌詞でしか、その存在意義を「推測」できない設計。
  * ここが本作最高潮のテーマ表現：「この世界の成り立ちを、人間は決して完全には知ることはない」

* **平易で自然な文章を心がける:**
  * NPCのセリフ、解説文、ゲーム内テキストは、難しい専門用語や複雑な表現を避ける。
  * 「～であり」「～すなわち」といった堅い言い回しより、「～なんだ」「～だから」のような会話的な文体を優先する。
  * 長い文は短く区切り、プレイヤー（特に若年層）がストレスなく読める流れを作る。
  * 同じ言葉の繰り返しや、響きが悪い表現は避けて、読む人の耳や心地よさを大切にする。
  * 「すべてを説明する」のではなく「余白を感じさせる」ために、余計な形容詞は削ぎ落とす。
