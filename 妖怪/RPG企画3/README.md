**【プロジェクト・コア・フィロソフィー】**
* **タイトル:** アシブネノミコト —泥と星の神話—
* **テーマ:** 「完璧な神の理」に対する、「不完全な人間の熱量」の肯定。泥（不完全な人間）が砕かれた星（カガセオ）の光で自らの傷を繋ぎ、天の理を砕く物語。
* **結末の哲学（永遠の拒絶）:** コノハナサクヤヒメとイワナガヒメの神話の脱構築。主人公は神の血を引く「新たな支配者（帝）」になるのではなく、傷も変化も拒む永遠（凍結した理／イワナガ）を捨て、寿命と劣化（サクヤ）を選び取った「最初の『ただの人』」となる。
* **デザイン原則:**
	* 神（敵・システム）は、傷を恐れ、静謐で端正な予測（エントロピーの凍結）を行う。
  * 人（味方・プレイヤー）は、地に堕ちた星の砂（金継ぎ）を用いて傷を歴史として引き受け、摩擦やエラー（ノイズ）で神の理を砕く。
* **すべての仕様は、「効率的な消費」ではなく「泥臭い修復と継承」をプレイヤーに体感させるために存在する。**

## 運用ルール

- すべてのファイル冒頭に同一の「プロジェクト・コア・フィロソフィー」を置く。
- 仕様の重複記述を避け、各ファイルの責務境界を維持する。
  * 例: 神社建立については 02 に計算ロジックと条件を、03 に操作感と UI フローを記述する。
- 物語（01）・ルール（02）・体験設計（03）・数値（04）を分離し、更新漏れを防ぐ。
- **ビジュアル定義:** キャラクターやUIの美術設定、AI生成用プロンプトは `05_Art_and_Visual_Design.md` に集約する。
- **造形美の優先:** キャラクターには泥臭い設定があっても、顔立ちと骨格は万人受けする圧倒的な美しさ・華やかさを担保する。無骨さや傷はあくまでも「美しいキャンバスの上の装飾」として扱い、醜さに転ぼうとする語彙を避ける。
- 新規決定事項を追記する際は、まず `README.md` の反映状況を更新する。
- **グリッドサイズ:** 初期神社敷地は 10×10 程度のブロックで表現。細部パーツの配置容量を早期に確保するため。
- **ワープ演出の一貫性:** マップ上のひび割れ／漆による結合表現は各ドキュメントに反映し、移動時のアニメーション・音響が物語的シンボルとなるよう統一する。
